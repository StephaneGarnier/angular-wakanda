Contribute

Angular-Wakanda relies on several Open Source projects available on Github.

Wakanda/angular-wakanda

It is the production repository for the connector. Bower and npm take version on it. Do not make any pull request on this repository.

Wakanda/sources-angular-wakanda

It is the development repository. It contains sources for the connector, but also a Wakanda solution, unit tests, end-to-end tests and a demo application.
As all of this is quite heavy, we did not want to include all of this stuff on the distribution repository, that's why we split them up. Pull requests must be opened on this repository.

How to contribute ?

If you want to submit a feature on the Angular-Wakanda connector :

  1. Fork the sources-angular-wakanda repository
  2. Made a branch from the actual develop branch
  3. Implement your feature or fix
  4. Implement unit-tests for your feature, or update existing one if necessary
  5. Open a pull request to merge your branch on the develop branch of sources-angular-wakanda repository. Describe from a user point of view what you feature does, and how.
  6. Discuss with Wakanda team and improve your PR if necessary
Thanks for contributing ! ;-)

Wakanda/generator-angular-wakanda

It is the yeoman generator repository. It is based on yeoman/generator-angular repository, and you also can use any command of generator-angular on a project generated with our connector.

Wakanda/wakanda-issues

If you find a bug in the source code or a mistake in the documentation, or if you need a new feature, you can submit an issue to this unique Wakanda Issues Repository.
